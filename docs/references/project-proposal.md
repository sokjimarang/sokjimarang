### 1.2 문제의 본질

**왜 사람들은 속는가?**

보이스피싱은 정보 부족이 아니라 **심리적 취약성**을 공격한다.

1. **공포 유발**: "범죄에 연루되었다", "아들이 사고를 당했다"
2. **시간 압박**: "지금 당장 조치하지 않으면 처벌받는다"
3. **권위 활용**: "검찰입니다", "금융감독원입니다"
4. **고립 유도**: "이 사실을 타인에게 알리면 증거인멸이다"

**왜 기존 교육은 실패하는가?**

| 기존 교육 방식 | 한계 |
| --- | --- |
| 강의/세미나 | 일방적 전달, 체험 없음 |
| 영상/웹툰 | 수동적 시청, 실제 상황과 괴리 |
| 문자/팝업 알림 | 무시되기 쉬움, 경각심 형성 안됨 |

공통 문제: **"나는 안 속아"라는 과신** → 직접 경험하기 전까지는 자신의 취약성을 인지하지 못함

### 1.3 해결 가설

> "안전한 환경에서 속아보는 경험이 실제 상황에서의 대응력을 높인다"
> 

이 가설의 전제:

1. 체험 기반 학습이 청각적 학습보다 기억에 오래 남는다
2. 한 번 경험한 패턴은 재노출 시 인지하기 쉽다
3. 안전한 실패 경험이 경각심을 형성한다

## 2. 솔루션

### 2.1 서비스 정의

**"속지마랑"**: AI가 보이스피싱 사기범 역할을 수행하여, 사용자가 안전한 환경에서 보이스피싱을 체험하는 서비스

### 2.2 핵심 가치

| 가치 | 정의 |
| --- | --- |
| 체험 기반 학습 | 듣는 것이 아닌 겪는 것 |
| 실시간 상호작용 | 내 반응에 따라 변하는 AI 대화 |
| 안전한 실패 | 실제 피해 없이 속아보는 경험 |
| 즉각적 피드백 | 어디서 속았는지, 어떻게 대응해야 하는지 |

### 2.3 타겟 사용자

**1차 타겟 (실제 사용자)**: 50대 이상

- 보이스피싱 피해 주요 대상 (전체의 53%)
- 전화 통화에 익숙
- 앱 기본 조작 가능, 복잡한 UI는 어려움

**2차 타겟 (설치/안내자)**: 30~40대 자녀 세대

- 부모님 대신 앱 설치 및 초기 설정
- 서비스 안내 및 권유
- 추후 유료화 시 결제 주체

## 3. 핵심 기능

### 3.1 MVP 기능 범위

| 기능 | 포함 여부 | 비고 |
| --- | --- | --- |
| 훈련 시작 (버튼 → AI 전화) | ✓ |  |
| AI 보이스피싱 시뮬레이션 | ✓ | 4개 시나리오 |
| 맞춤형 컨텍스트 입력 | ✓ | 연령대, 지역, 가족구성 |
| 디브리핑 (음성 + 화면) | ✓ |  |
| 실제 전화번호 발신 | ✗ | MVP 이후 검토 |
| Voice Cloning | ✗ | 법적 리스크, MVP 이후 검토 |
| iOS 앱 | ✗ | MVP 이후 |

### 3.2 시나리오 유형

| 번호 | 유형 | 설명 |
| --- | --- | --- |
| 1 | 검찰/경찰 사칭 | "서울중앙지검입니다. 범죄에 연루되었습니다" |
| 2 | 금융기관 사칭 | "OO은행입니다. 이상 거래가 감지되었습니다" |
| 3 | 가족 납치/사고 사칭 | "아들입니다... 사고가 났어요" |
| 4 | 택배/정부지원금 사칭 | "미수령 택배가 있습니다" |

## 4. 기술 방향

### 4.1 아키텍처

```
┌─────────────────────────────────────────────┐
│              속지마랑 아키텍처               │
│                                             │
│  📱 React Web App                           │
│     (React Native WebView로 앱 래핑)        │
│     │                                       │
│     │ WebSocket                             │
│     ▼                                       │
│  ┌─────────────────────────────────────┐   │
│  │     ElevenLabs Conversational AI     │   │
│  │                                      │   │
│  │  ASR(Scribe) → LLM → TTS(Turbo v2.5) │   │
│  │         ↑                            │   │
│  │    System Prompt                     │   │
│  │    + 시나리오 + 컨텍스트             │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘

```

### 4.2 기술 스택

| 레이어 | 기술 |
| --- | --- |
| 모바일 앱 | React (Web) + React Native WebView |
| Voice AI | ElevenLabs Conversational AI |
| ASR (음성→텍스트) | ElevenLabs Scribe |
| LLM | Gemini 2.5 Flash |
| TTS (텍스트→음성) | ElevenLabs Turbo v2.5 |

### 4.3 예상 비용

| 구성요소 | 비용/분 |
| --- | --- |
| ElevenLabs Agents | ~$0.08 |
| LLM (Gemini 2.5 Flash) | ~$0.01 |
| **총합** | **~$0.09/분** |

*5분 통화 기준: 약 $0.45 (≈650원)*

## 5. 사용자 경험 흐름

### 5.1 전체 흐름

```
앱 설치 → 초기 설정 → 메인 화면 → 훈련 시작 →
AI 전화 수신 → 통화 진행 → 디브리핑(음성) → 디브리핑(화면)
```

### 5.2 화면별 정의

| 화면 | 목적 | 주요 요소 |
| --- | --- | --- |
| 온보딩 | 서비스 소개, 동의 획득 | 스와이프 카드 3장, 이용약관 체크박스 |
| 메인 | 훈련 진입점 | "훈련 시작" 버튼, 시나리오 선택, 훈련 기록 |
| 컨텍스트 입력 | 맞춤형 시나리오용 정보 | 연령대, 거주지역, 가족구성 선택 |
| 통화 | 시뮬레이션 진행 | 통화 UI, "훈련 종료" 버튼 |
| 디브리핑 | 결과 및 교육 | 진행 단계, 탐지 포인트, 올바른 대응 |
